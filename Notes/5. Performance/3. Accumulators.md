## The Technique

An **accumulator** is an extra parameter that builds up the result incrementally as recursion progresses, instead of combining results on the way back up. This turns non-tail-recursive functions into tail-recursive ones.

## The General Framework

For a function `f`, we want to find:
- An accumulator-based helper `fAux xs acc` satisfying: `fAux xs acc = acc ⊕ f xs`
- An operator `⊕` that combines the accumulated result with the remaining work
- An initial value `init` that is the **left-neutral element** of `⊕` (so `init ⊕ f xs = f xs`)

Then: `fTR xs = fAux xs init`

## Derivation: Length

**Original:**
```lean
def len [] = 0
def len (_ :: xs) = 1 + len xs
```

**Choose:** `⊕ = (+)`, `init = 0` (since `0 + x = x`)

**Identity:** `lenAux xs acc = acc + len xs`

**Derive by cases:**

| Case | Derivation |
|---|---|
| `xs = []` | `lenAux [] acc = acc + 0 = acc` |
| `xs = _ :: xs'` | `lenAux (_ :: xs') acc = acc + (1 + len xs') = (acc + 1) + len xs' = lenAux xs' (acc + 1)` |

**Result:**
```lean
def lenAux (xs : List α) (acc : Nat) : Nat :=
  match xs with
  | [] => acc
  | _ :: xs => lenAux xs (acc + 1)

def lenTR (xs : List α) : Nat := lenAux xs 0
```

**Complexity:** O(n) — same as before, but now tail-recursive.

## Derivation: Reverse

**Original (O(n²)):**
```lean
def rev [] = []
def rev (x :: xs) = rev xs ++ [x]
```

**Choose:** `⊕ = flip (++)` (i.e., `acc ⊕ y = y ++ acc`), `init = []`

**Identity:** `revAux xs acc = (rev xs) ++ acc`

**Derive by cases:**

| Case | Derivation |
|---|---|
| `xs = []` | `revAux [] acc = [] ++ acc = acc` |
| `xs = x :: xs'` | `revAux (x :: xs') acc = (rev xs' ++ [x]) ++ acc = rev xs' ++ (x :: acc) = revAux xs' (x :: acc)` |

**Result:**
```lean
def revAux (xs acc : List α) : List α :=
  match xs with
  | [] => acc
  | x :: xs => revAux xs (x :: acc)

def revTR (xs : List α) : List α := revAux xs []
```

**Complexity:** O(n) — each step does one cons operation (O(1)). Down from O(n²).

### Execution Trace

```
revAux [1,2,3,4,5] []
= revAux [2,3,4,5] [1]
= revAux [3,4,5]   [2,1]
= revAux [4,5]     [3,2,1]
= revAux [5]       [4,3,2,1]
= revAux []        [5,4,3,2,1]
```

Each step moves one element from the input to the front of the accumulator — effectively reversing the list.

## Derivation: Append

**Original:**
```lean
def app [] ys = ys
def app (x :: xs) ys = x :: app xs ys
```

### First Attempt (fails)

`⊕ = (++)`, identity `appAux xs acc = acc ++ xs ++ ys` — leads to `acc ++ [x]` in the recursive case, which is still O(n). No improvement.

### Working Approach

Process `xs` in reverse, build the result from right to left.

**New identity:** `appAux xs acc = (rev xs) ++ acc`, with `init = ys`

| Case | Derivation |
|---|---|
| `xs = []` | `appAux [] acc = acc` |
| `xs = x :: xs'` | `appAux (x :: xs') acc = rev xs' ++ (x :: acc) = appAux xs' (x :: acc)` |

**Result:**
```lean
def appAux (xs acc : List α) : List α :=
  match xs with
  | [] => acc
  | x :: xs => appAux xs (x :: acc)

def appTR (xs ys : List α) : List α := appAux xs.reverse ys
```

Note: `appAux` is identical to `revAux` — appending is just reversing the first list onto the second.

## Benchmark Results

| Function | Naive | Accumulator-based | Standard library |
|---|---|---|---|
| `length` | O(n), higher constant | O(n), lower constant | O(n) ≈ accumulator |
| `append` | O(n), higher constant | O(n), lower constant | O(n) ≈ accumulator |
| `reverse` | **O(n²)** | **O(n)** | O(n) ≈ accumulator |

The accumulator-based versions match standard library performance.

## When Accumulators Help

Accumulators help when the original function does **work after the recursive call** (non-tail position). By shifting that work into the accumulator update, the recursive call moves to tail position.

| Original pattern | Problem | Accumulator fix |
|---|---|---|
| `1 + f xs` | Addition after call | `f xs (acc + 1)` |
| `f xs ++ [x]` | Append after call (O(n)) | `f xs (x :: acc)` |
| `x :: f xs` | Cons after call | `f xs (x :: acc)`, reverse at end if needed |

> [!tip] The Creative Part
> Finding the right `⊕` and `init` requires insight into the specific problem. The framework narrows the search, but the "aha moment" — like realizing reverse needs `flip (++)` — comes from understanding what the accumulator represents at each step.

> [!summary]
> Accumulators transform non-tail-recursive functions into tail-recursive ones by building the result incrementally in an extra parameter. The derivation follows a systematic framework: choose `⊕` and `init`, verify the identity holds for base and recursive cases, and the implementation falls out.
