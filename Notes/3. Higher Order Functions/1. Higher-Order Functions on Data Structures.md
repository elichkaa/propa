## The Problem: Repeated Structure

Many list functions share the same recursive skeleton — only the per-element operation differs:

```lean
-- hash each string
def hashCodes (lst : List String) : List UInt64 :=
  match lst with
  | [] => []
  | hd :: tl => hd.hash :: hashCodes tl

-- negate each bool
def negate (lst : List Bool) : List Bool :=
  match lst with
  | [] => []
  | hd :: tl => (!hd) :: negate tl
```

Same recursion, same base case, only the operation on `hd` changes. Duplicating this skeleton for every operation is tedious and error-prone.

## The Solution: Parameterize the Operation

A **higher-order function** takes the varying part as a function argument, keeping the shared structure in one place.

### map — transform each element

```lean
def map (f : α → β) (lst : List α) : List β :=
  match lst with
  | [] => []
  | hd :: tl => f hd :: map f tl
```

Now the duplicated functions are one-liners:

```lean
def hashCodes := List.map String.hash
def negate    := List.map (!·)
def round     := List.map Float.round
```

### filter — keep elements that satisfy a predicate

```lean
def filter (p : α → Bool) (lst : List α) : List α :=
  match lst with
  | [] => []
  | hd :: tl => if p hd then hd :: filter p tl else filter p tl
```

```lean
List.filter (fun x => x % 2 == 0) [1, 2, 3, 4]  -- [2, 4]
List.filter (fun s => s != "") ["hi", "", "bye"]  -- ["hi", "bye"]
```

### zipWith — combine two structures element-wise

```lean
def zipWith (f : α → β → γ) (xs : List α) (ys : List β) : List γ :=
  match xs, ys with
  | [], _ | _, [] => []
  | x :: xs, y :: ys => f x y :: zipWith f xs ys
```

Stops at the shorter list. Plain `zip` is just `zipWith (·, ·)`.

## The Same Pattern Works for Any Data Structure

Higher-order functions aren't limited to lists. For any recursive type, you can define `map`, `filter`, `zipWith` by following the type's recursive structure:

```lean
inductive BinaryTree (α : Type) where
  | empty
  | leaf (elem : α)
  | node (left : BinaryTree α) (right : BinaryTree α)

def BinaryTree.map (f : α → β) : BinaryTree α → BinaryTree β
  | empty       => empty
  | leaf x      => leaf (f x)
  | node l r    => node (l.map f) (r.map f)
```

The shape of the function mirrors the shape of the data — one case per constructor, recursive calls on recursive fields.

## Summary of Core Higher-Order Functions

| Function | What it does | Type (for lists) |
|---|---|---|
| `map` | Transform each element | `(α → β) → List α → List β` |
| `filter` | Keep elements matching a predicate | `(α → Bool) → List α → List α` |
| `zipWith` | Combine two structures element-wise | `(α → β → γ) → List α → List β → List γ` |

> [!tip] Recognizing the Pattern
> If you're writing a recursive function and the only thing that varies from a previous function is a small operation — extract that operation into a parameter. You've just discovered a higher-order function.
