## The Problem

`List.map` applies a pure function to each element:

```
List.map : (α → β) → List α → List β
```

But what if the function can fail? If `f : α → Except E β`, then `xs.map f` gives `List (Except E β)` — a list where each ele ment *individually* might have failed.

**What you actually want:** run `f` on every element, and if *any* call fails, the whole thing fails. Success means every element succeeded.

```
List (Except E β)       ← what map gives you (useless)
Except E (List β)       ← what you actually want
```

## Effectful Map

```lean
def List.mapExcept (f : α → Except ε β) (xs : List α) : Except ε (List β) := do
  match xs with
  | [] => return []
  | x :: xs' =>
    let y ← f x                    -- fails here → everything fails
    let ys ← mapExcept f xs'
    return y :: ys
```

The `←` on each recursive call ensures errors **short-circuit** — the first failure stops everything.

Applied to argument parsing:

```lean
def parseArgs (args : List String) : Except Error (List Argument) :=
  args.mapExcept parseArg
```

One function call — you either get all parsed arguments or the first error.

## The General Pattern: mapM

This pattern isn't specific to `Except`. The standard library provides `List.mapM`, which works for **any effect** — `IO`, `Except`, `Option`, or anything with monadic structure:

```
List.mapM : (α → m β) → List α → m (List β)
```

Where `m` can be `IO`, `Except E`, `Option`, etc.

Whenever you have a list and a function that returns an effectful result, `mapM` threads the effect through the whole list. The same pattern exists for other operations:

| Function | Pure version | Effectful version |
|---|---|---|
| map | `List.map` | `List.mapM` |
| filter | `List.filter` | `List.filterM` |
| fold | `List.foldl` | `List.foldlM` |

The **`M` suffix** is the convention for "this is the effectful version."

## Composing Different Effects

A real `main` function often combines multiple effect types. A common pattern is to run a pure-but-fallible computation (returning `Except`) and then handle the result inside `IO`:

```lean
def main (args : List String) : IO Unit := do
  match parseArgs args with          -- parseArgs returns Except, not IO
  | Except.ok config =>
    match config.find? (·.isVar "name") with
    | some (Argument.var _ name) => greet name
    | _ => helloWorld
  | Except.error err =>
    IO.println s!"Error: {reprStr err}"
```

> [!summary] Composition Principle
> The boundary between `Except` and `IO` is explicit — you pattern match to cross it. Each effect handles one concern (failure vs I/O), and you compose them at the edges rather than tangling them together.
