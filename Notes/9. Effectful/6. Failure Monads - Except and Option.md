
Two monads that model computations which can fail. They differ only in whether the failure carries information.

## Except Monad

`Except ε α` either succeeds with a value of type `α` or fails with an error of type `ε`.

### Monad Instance

```lean
instance : Monad (Except ε) where
  pure x := Except.ok x
  bind x f := match x with
    | Except.ok v    => f v                -- success: continue
    | Except.error e => Except.error e     -- failure: propagate, skip f
```

**`bind` is where error propagation happens.** If the first computation failed, the second one never runs. This is why `let x ← ...` in a `do` block automatically escalates errors.

### Helper Operations

| Operation | Type | Meaning |
|---|---|---|
| `throw err` | `Except ε α` | Fail immediately with error `err` |
| `handle comp f` | `α` | Run `comp`; if it fails, recover using `f` |

```lean
def throw (err : ε) : Except ε α := Except.error err

def handle (comp : Except ε α) (handler : ε → α) : α :=
  match comp with
  | Except.ok v    => v
  | Except.error e => handler e
```

### Example: Parse and Add

Without monads — nested pattern matching that gets unwieldy fast:

```lean
def parseAndAdd_manual (s1 s2 : String) : Except String Int :=
  match parseInt s1 with
  | Except.ok n1 => match parseInt s2 with
    | Except.ok n2 => Except.ok (n1 + n2)
    | Except.error e => Except.error e
  | Except.error e => Except.error e
```

With the monad — flat and readable:

```lean
def parseAndAdd (s1 s2 : String) : Except String Int := do
  let n1 ← parseInt s1
  let n2 ← parseInt s2
  return n1 + n2
```

The `do` notation handles all the "if it failed, propagate" logic via `bind`.

## Option Monad

`Option α` is like `Except` but with no error information — it either succeeds (`some v`) or fails (`none`).

### Monad Instance

```lean
instance : Monad Option where
  pure x := some x
  bind x f := match x with
    | some v => f v
    | none   => none
```

Same propagation logic as `Except`: if any step yields `none`, the whole chain yields `none`.

### Example: Safe Division

```lean
def safeDiv (a b : Int) : Option Int :=
  if b = 0 then none else some (a / b)

def compute (x y z : Int) : Option Int := do
  let a ← safeDiv x y    -- none if y = 0
  let b ← safeDiv y z    -- none if z = 0
  return a + b
```

```
compute 10 2 5  =  some 7
compute 10 0 5  =  none      -- first division fails
compute 10 2 0  =  none      -- second division fails
```

### Recovery with orElse

```lean
def orElse (comp : Option α) (fallback : α) : α :=
  match comp with
  | some v => v
  | none   => fallback
```

## When to Use Which

| Monad | Failure carries info? | Use when... |
|---|---|---|
| `Option` | No (just `none`) | Simple presence/absence, lookup failures |
| `Except ε` | Yes (error of type `ε`) | You need to report *what* went wrong |
