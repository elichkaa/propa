## What Effect Does It Model?

**Mutable state** — computations can read and update a shared state value, and each subsequent computation sees the changes made by previous ones.

## Definition

```lean
structure State (σ α : Type) where
  run : σ → (α × σ)
```

A `State σ α` is a function that takes an initial state, does something, and returns a result **plus a new state**. The state is threaded through the chain.

## Monad Instance

```lean
instance : Monad (State σ) where
  pure x := ⟨fun s => (x, s)⟩             -- return x, state unchanged
  bind m f := ⟨fun s =>
    let (v, s') := m.run s                  -- run first, get updated state s'
    (f v).run s'                            -- run second with s', not s
  ⟩
```

Key difference from Reader: **`bind` passes the *updated* state `s'` to the second computation**, not the original `s`. State flows forward.

## Helper Operations

| Operation | Type | Purpose |
|---|---|---|
| `get` | `State σ σ` | Read the current state |
| `set s` | `State σ Unit` | Replace the state entirely |
| `modify f` | `State σ Unit` | Apply `f` to the current state |

```lean
def get : State σ σ := ⟨fun s => (s, s)⟩
def set (s : σ) : State σ Unit := ⟨fun _ => ((), s)⟩
def modify (f : σ → σ) : State σ Unit := do
  let s ← get
  set (f s)
```

Note that `modify` itself is defined using `do` notation — it's a monadic computation composed from `get` and `set`.

## Example: Simple Counter

```lean
def tick  : State Nat Unit := modify (· + 1)
def reset : State Nat Unit := set 0

def ticker : State Nat (Nat × Nat) := do
  tick; tick
  let n1 ← get       -- 2

  reset
  [0:5].forM (fun _ => tick)
  let n2 ← get       -- 5

  return (n1, n2)
```

Running: `ticker.run 0` produces `((2, 5), 0)` — result is `(2, 5)`, final state is `0` after the last reset.

## Example: Complex State — Lap Timer

The state can be any structure, not just a number:

```lean
structure LapState where
  current : Nat
  total   : Nat
  laps    : Nat
  bestLap : Option Nat
  avgLap  : Float
```

Operations update specific fields:

```lean
def tick : State LapState Unit :=
  modify (fun s => { s with current := s.current + 1, total := s.total + 1 })

def lap : State LapState Unit := do
  let s ← get
  let best := match s.bestLap with
    | some b => Nat.min b s.current
    | none   => s.current
  set ⟨0, s.total, s.laps + 1, some best, s.total.toFloat / (s.laps + 1).toFloat⟩
```

A race simulation:

```lean
def race := do
  [0:10].forM (fun _ => tick); lap    -- lap 1: 10 ticks
  [0:12].forM (fun _ => tick); lap    -- lap 2: 12 ticks
  [0:8].forM  (fun _ => tick); lap    -- lap 3: 8 ticks
```

Running `race.run {}` gives final state with `total = 30`, `laps = 3`, `bestLap = 8`, etc.

## Reader vs State

| | Reader | State |
|---|---|---|
| Environment | Read-only, shared | Read-write, threaded |
| `bind` passes | Same environment to both | Updated state to second |
| Use case | Configuration, context | Counters, accumulators, simulations |

> [!summary]
> The State monad simulates mutable variables in a pure setting. Each "mutation" actually produces a new state value that gets threaded through via `bind`. The code *looks* imperative but *is* purely functional.
