## The Situation

Real programs need multiple effects simultaneously. A function might need to:
- Read/modify state **and** throw errors
- Perform I/O **and** track state
- Handle errors **and** log messages **and** read configuration

But monadic functions must live in a **single monad**. You can't mix `Except String` and `StateM Int` in one `do` block:

```lean
-- Does NOT compile: parseInt is in Except, runningSum is in StateM
def process (input : String) : ??? := do
  let i ← parseInt input       -- Except String Int
  runningSum i                  -- StateM Int Unit
```

There's no type that satisfies both — the two monads are incompatible.

## Brute Force: Write a Custom Combined Monad

You can manually define a monad that handles both effects:

```lean
structure ExceptStateM (σ ε α : Type) where
  run : σ → Except ε (α × σ)
```

This is a function that takes an initial state, and either fails with an error or returns a result paired with the updated state.

You then implement `Monad`, plus custom `throw` and `modify`:

```lean
instance : Monad (ExceptStateM σ ε) where
  pure a := ⟨fun s => Except.ok (a, s)⟩
  bind f g := ⟨fun s =>
    match f.run s with
    | Except.error e => Except.error e
    | Except.ok (a, s') => (g a).run s'⟩
```

This works, but the boilerplate is painful:
- New `structure` for every combination of effects
- New `Monad` instance each time
- Can't reuse `throw` or `modify` from the standard library — must redefine them
- Adding a third effect (e.g., IO) means starting over

> [!warning] This Doesn't Scale
> With n effect types, you'd need up to n! custom combined monads. The solution: **monad transformers**.
