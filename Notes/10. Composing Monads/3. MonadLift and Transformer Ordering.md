## Lifting Operations Between Layers

In a transformer stack like `StateT Int (ExceptT String IO)`, operations belong to different layers:

| Operation | Native layer |
|---|---|
| `modify` | `StateT` (outermost) |
| `throw` | `ExceptT` (middle) |
| `IO.println` | `IO` (innermost) |

To use an inner operation in the outer monad, you need to **lift** it — translate it through the layers. `MonadLift` provides this:

```lean
class MonadLift (m : Type → Type) (n : Type → Type) where
  monadLift : m α → n α
```

Each transformer defines how to lift the wrapped monad into itself:

```lean
-- Lifting m into StateT σ m: run the m action, pass state through unchanged
instance [Monad m] : MonadLift m (StateT σ m) where
  monadLift ma := ⟨fun s => do let a ← ma; pure (a, s)⟩

-- Lifting m into ExceptT ε m: run the m action, wrap result in Ok
instance [Monad m] : MonadLift m (ExceptT ε m) where
  monadLift ma := ⟨do let a ← ma; pure (Except.ok a)⟩
```

Lean chains these automatically — `IO.println` in a `StateT Int (ExceptT String IO)` context gets lifted through both layers.

## Transformer Ordering Matters

`StateT σ (Except ε)` and `ExceptT ε (StateM σ)` are **not the same**:

| Stack | On error... | Semantics |
|---|---|---|
| `StateT σ (Except ε)` | State is **discarded** | Error aborts everything, state rolled back |
| `ExceptT ε (StateM σ)` | State is **preserved** | Error returned, but state changes kept |

```
StateT σ (Except ε) α  =  σ → Except ε (α × σ)    -- error means no state
ExceptT ε (StateM σ) α  =  σ → (Except ε α) × σ   -- state always returned
```

### How to Choose

- **Outer transformer** = the effect you interact with "first" at the API level
- **Inner monad** = the effect that is more "fundamental" or always present
- Ask: "if an error occurs, should state changes be visible?" If no → state outside. If yes → except outside.

> [!important]
> The order of transformers is a **design decision** that affects program semantics. Always think about what should happen to each effect when another effect triggers (e.g., what happens to state on error? what happens to errors on exit?).
