## Beta Reduction

The single computation rule of lambda calculus: applying a function replaces the parameter with the argument throughout the body.

```
(λx. expr) arg  →  expr[x := arg]
```

"Evaluate a function call by substituting the argument for the parameter."

## Substitution Rules

Substitution `expr[x := s]` means "replace every free occurrence of `x` in `expr` with `s`." Three cases based on what `expr` is:

### Case 1: Variable

```
y[x := s]  =  s     if y == x    (match — replace)
y[x := s]  =  y     if y ≠ x     (no match — leave alone)
```

```
(λx. x) s  →  x[x := s]  →  s
(λx. y) s  →  y[x := s]  →  y
```

### Case 2: Application

Recurse into both function and argument:

```
(f a)[x := s]  =  (f[x := s]) (a[x := s])
```

```
(λx. x x) s  →  (x x)[x := s]  →  s s
```

### Case 3: Lambda (the tricky case)

`(λy. body)[x := s]` has three sub-cases:

| Condition | Action | Why |
|---|---|---|
| `y == x` | Return unchanged | `y` **shadows** `x` — `x` is no longer free inside |
| `y ∉ freeVars(s)` | Recurse into body | Safe — no name collision |
| `y ∈ freeVars(s)` | **Rename** `y` first, then recurse | Prevents **variable capture** |

**Variable capture example:**

```
(λy. x)[x := y]  →  λy. y   ← WRONG! (became identity function)

Fix: rename y → z first
(λy. x)  →  (λz. x)  →  (λz. x)[x := y]  →  λz. y   ← CORRECT ✅
```

### Implementation

```lean
partial def substitute (e : Expression) (name : String) (value : Value) : Expression :=
  let rec subst : Expression → Expression
    | .var n        => if n == name then value.asExpression else .var n
    | .lambda x body =>
        if x == name then .lambda x body              -- shadowed
        else if value.hasFreeVar x then panic! "..."   -- capture (needs renaming)
        else .lambda x (subst body)                    -- safe to recurse
    | .apply f a    => .apply (subst f) (subst a)
    | other         => -- recurse into sub-expressions
  subst e
```

> [!warning] Variable Capture
> Forgetting to handle the capture case is one of the most common bugs in language implementations. Always check whether the lambda's bound variable appears free in the replacement term.
