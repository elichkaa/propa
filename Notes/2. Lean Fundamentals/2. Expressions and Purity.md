## Everything is an Expression

In functional programming, **every construct produces a value**. There are no "statements" that just perform actions — only expressions that compute results.

This is why `if-then-else` always requires an `else` branch: it must produce a value in all cases.

```lean
def absolute (x : Int) : Nat :=
  if x < 0 then (-x).toNat
  else x.toNat
```

## Expressions Compose Freely

Because everything is an expression, you can nest constructs anywhere a value is expected:

```lean
-- if-then-else as a function argument
add (if x > 0 then 10 else 20) (if y > 0 then 30 else 40)

-- if-then-else as a condition
if (if true then false else true) then "yes" else "no"
```

This composability is a direct consequence of the expression-based design. In imperative languages, `if` is often a statement that can't be placed inside another expression.

## Pure Functions

A **pure function** has no side effects — its output depends only on its inputs:

- Same input → always same output
- No printing, no file access, no global state modification

This makes functions predictable, testable, and safe to refactor. You can always replace a function call with its result (**equational reasoning**) without changing behavior.

## Extracting Subexpressions

Because everything is an expression, refactoring is straightforward — replace any subexpression with a function call:

```lean
-- Before: duplicated logic
def describe (x : Int) : String :=
  if x < 0 then "negative " ++ if x % 2 == 0 then "even" else "odd"
  else "positive " ++ if x % 2 == 0 then "even" else "odd"

-- After: extracted into a function
def parity (x : Int) : String :=
  if x % 2 == 0 then "even" else "odd"

def describe (x : Int) : String :=
  (if x < 0 then "negative " else "positive ") ++ parity x
```

> [!tip] Expression Languages
> Languages where programs are built entirely from expressions (no statements) are called **expression languages**. This design enables the deep composability and refactorability that functional programming is known for.
