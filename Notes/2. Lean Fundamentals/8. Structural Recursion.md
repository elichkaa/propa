## The Template

Every structurally recursive function on a data type follows the same shape:

1. **Base case(s):** handle non-recursive constructors — return a value directly
2. **Recursive case(s):** handle recursive constructors — process the current element, make a recursive call on the sub-structure, combine the results

```
match value with
| <base constructor>      => <base result>
| <recursive constructor> => <combine current with recursive call on sub-part>
```

## Examples on Lists

The template is the same every time — only the "combine" step changes:

| Function | Base (`[]`) | Combine (`x :: xs`) |
|---|---|---|
| `length` | `0` | `1 + length xs` |
| `sum` | `0` | `x + sum xs` |
| `append xs ys` | `ys` | `x :: append xs ys` |
| `map f` | `[]` | `f x :: map f xs` |

```lean
def length (l : List α) : Nat :=
  match l with
  | [] => 0
  | _ :: tl => 1 + length tl

def append (xs ys : List α) : List α :=
  match xs with
  | [] => ys
  | x :: xs => x :: append xs ys
```

## Examples on Natural Numbers

Same template, different type:

```lean
def add (m n : Nat) : Nat :=
  match m with
  | zero   => n                     -- base: 0 + n = n
  | succ m' => succ (add m' n)      -- recursive: (m+1) + n = (m + n) + 1

def mul (m n : Nat) : Nat :=
  match m with
  | zero    => zero                 -- base: 0 × n = 0
  | succ m' => add n (mul m' n)     -- recursive: (m+1) × n = n + (m × n)
```

## Multiple Recursive References

When a data type has constructors with multiple sub-structures, make a recursive call on each:

```lean
def turnCount (r : Route) : Nat :=
  match r with
  | destination _ => 0
  | unreachable   => 0
  | turn _ rest       => 1 + turnCount rest
  | sequence r1 r2    => turnCount r1 + turnCount r2
  | alternative r1 r2 => Nat.min (turnCount r1) (turnCount r2)
```

## Simultaneous Recursion

Recurse on multiple arguments at the same time when both shrink together:

```lean
def take (n : Nat) (l : List α) : List α :=
  match n, l with
  | 0, _          => []
  | _, []         => []
  | n + 1, x :: xs => x :: take n xs
```

## Why Structural Recursion Matters

- **Totality:** each recursive call is on a strictly smaller sub-structure, so termination is guaranteed. Lean can verify this automatically.
- **Predictability:** the function's shape mirrors the data's shape — easy to read and reason about.
- **Universality:** works for any recursive data type — lists, trees, natural numbers, custom types.

> [!summary]
> Structural recursion = follow the shape of the data. One branch per constructor, recursive calls only on sub-parts. Lean proves termination automatically.
