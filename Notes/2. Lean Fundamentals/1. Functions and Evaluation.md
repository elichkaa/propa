## Evaluating Expressions

`#eval` evaluates an expression and prints the result. Useful for quick exploration:

```lean
#eval 3 + 4             -- 7
#eval "Hello, " ++ "world!"
#eval true && false      -- false
```

## Defining Functions

Functions are defined with `def`. Functions without parameters are constants:

```lean
def zero : Int := 0
def add (x y : Int) : Int := x + y
```

## Function Application

Arguments are separated by **spaces**, not parentheses. Parentheses are only for grouping:

```lean
#eval add 5 10           -- 15
#eval add (2 * 3) (4 * 5)  -- 26
```

This is different from C-style `add(5, 10)`. In Lean, parentheses around multiple values create a **tuple**, which is a single value — not multiple arguments:

```lean
-- add (10, 20)  -- type error: tuple ≠ two arguments
add 10 20        -- correct
```

## Pipes

The pipe operator `|>` passes the left-hand result as the **last argument** to the right-hand function. Enables readable left-to-right chaining:

```lean
#eval [1, 2, 3, 4, 5]
  |> List.map (fun x => x * 2)
  |> List.filter (fun x => x > 5)
  |> List.sum
```

Each line transforms the result of the previous one. You can comment out any line (except the first) to see intermediate results.
