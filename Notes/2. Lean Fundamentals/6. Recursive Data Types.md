## What Makes a Type Recursive

A data type is **recursive** when one of its constructors refers back to the type itself. This allows representing structures of arbitrary size.

## Type Parameters (Generics)

Types can be parameterized over other types. `List α` means "a list of elements of type `α`":

```lean
inductive List (α : Type) where
  | nil
  | cons (hd : α) (tail : List α)
```

`List Nat` substitutes `Nat` for `α` everywhere, so `cons` becomes `Nat → List Nat → List Nat`. All elements must share the same type — the list is **homogeneous**.

## Classic Examples

### Lists — zero or more elements in sequence

```lean
inductive List (α : Type) where
  | nil                              -- base case: empty
  | cons (hd : α) (tail : List α)   -- recursive case: element + rest
```

### Natural numbers — zero or successor of another natural

```lean
inductive Nat where
  | zero                    -- base case
  | succ (n : Nat)          -- recursive case
```

`succ (succ (succ zero))` represents 3. Arithmetic is defined by recursion on this structure.

### Routes — multiple base cases and multiple recursive cases

```lean
inductive Route where
  | destination (name : String)            -- base case 1
  | unreachable                            -- base case 2
  | turn (dir : String) (rest : Route)     -- recursive case (1 sub-route)
  | sequence (r1 : Route) (r2 : Route)     -- recursive case (2 sub-routes)
  | alternative (r1 : Route) (r2 : Route)  -- recursive case (2 sub-routes)
```

## General Shape

A recursive data type can have:
- **Any number of base cases** (non-recursive constructors)
- **Any number of recursive cases**, each containing **one or more** recursive references

The key constraint: every recursive reference must appear as an argument to a constructor, ensuring values are always finite and well-founded.

> [!tip] Design Principle
> The structure of a recursive data type dictates the structure of every function that operates on it (see [[8. Structural Recursion]]).
