## Basics

Lists are ordered, homogeneous collections — all elements must have the same type.

```lean
[1, 2, 3, 4, 5]              -- List Nat
["apple", "banana", "cherry"] -- List String
```

## Construction

Under the hood, a list is either:
- **Empty**: `[]` (also called `nil`)
- **An element prepended to another list**: `x :: xs` (also called `cons`)

```lean
1 :: 2 :: 3 :: []    -- same as [1, 2, 3]
```

`::` is right-associative, so `1 :: 2 :: 3 :: []` means `1 :: (2 :: (3 :: []))`.

## Common Operations

```lean
[1, 2] ++ [3, 4]                    -- concatenation: [1, 2, 3, 4]
List.range 5                         -- [0, 1, 2, 3, 4]
List.range' 10 5                     -- [10, 11, 12, 13, 14]
List.map (fun x => x * 2) [1, 2, 3] -- [2, 4, 6]
List.filter (fun x => x > 2) [1, 2, 3, 4] -- [3, 4]
List.zip [1, 2] ["a", "b"]          -- [(1,"a"), (2,"b")]
```

## Type Homogeneity

All elements must share the same type. This is enforced at compile time:

```lean
-- ["apple"] ++ [3, 4]  -- type error: String ≠ Nat
```

## Why Lists Are Recursive

A list is defined in terms of itself: the tail of a `cons` is another list. This recursive structure is what enables pattern matching and structural recursion over lists — topics covered in [[7. Pattern Matching on Data]] and [[8. Structural Recursion]].
