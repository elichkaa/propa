## Purpose

`let` introduces a **local name** for a value within a specific scope. It improves readability, avoids repeated computation, and keeps definitions local rather than top-level.

```lean
def describe (x : Int) : String :=
  let sign := if x < 0 then "negative" else if x == 0 then "zero" else "positive"
  let parity := if x % 2 == 0 then "even" else "odd"
  sign ++ " " ++ parity
```

## Immutability

`let` bindings are **immutable** — once bound, a name cannot be reassigned. This is fundamental to functional programming: no variable ever changes, so you can always trust that a name refers to the same value throughout its scope.

## Inline Usage

Semicolons can separate `let` bindings on one line. `let` bindings can appear anywhere an expression is expected:

```lean
#eval let a := 3 * 7; add a a              -- 42
#eval add 10 (let b := 10 + 20; b * b)     -- 910
```

## let vs def

| | `let` | `def` |
|---|---|---|
| Scope | Local to enclosing expression | Top-level or namespace-level |
| Visibility | Only within its body | Anywhere that imports the module |
| Use | Intermediate computations | Reusable functions and constants |
