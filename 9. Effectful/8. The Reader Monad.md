## What Effect Does It Model?

**Shared read-only environment** — all computations in a chain can access the same configuration/context without passing it explicitly through every function call.

## Definition

```lean
structure Reader (ρ α : Type) where
  run : ρ → α
```

A `Reader ρ α` is just a function from an environment of type `ρ` to a result of type `α`. The monad machinery lets you avoid threading that argument manually.

## Monad Instance

```lean
instance : Monad (Reader ρ) where
  pure x := ⟨fun _ => x⟩               -- ignore environment, return x
  bind f1 f2 := ⟨fun env =>
    let v := f1.run env                  -- run first with env
    (f2 v).run env                       -- run second with same env
  ⟩
```

Key property: **`bind` passes the same environment to both computations**. The environment is never modified — it's shared, not threaded.

## Helper Operations

| Operation | Purpose |
|---|---|
| `read` | Retrieve the current environment |
| `withReader f comp` | Run `comp` with a modified environment (locally) |

```lean
def read : Reader ρ ρ := ⟨fun env => env⟩

def withReader (f : ρ → ρ) (comp : Reader ρ α) : Reader ρ α :=
  ⟨fun env => comp.run (f env)⟩
```

`withReader` doesn't permanently change the environment — it only affects the computation passed to it. Everything else still sees the original.

## Example: Session-Aware Requests

```lean
abbrev SessionID := String

def sendRequest (req : String) : Reader SessionID String := do
  let id ← read
  return s!"Sending '{req}' with session '{id}'"

def asAnonymous (f : Reader SessionID α) : Reader SessionID α :=
  withReader (fun _ => "") f

def mixedRequests : Reader SessionID String := do
  let r1 ← sendRequest "GET /data"           -- uses real session
  let r2 ← asAnonymous (sendRequest "POST")  -- uses empty session
  let r3 ← sendRequest "DELETE /data"         -- uses real session again
  return r1 ++ "\n" ++ r2 ++ "\n" ++ r3
```

Running: `mixedRequests.run "session_12345"`

The session ID flows through without being passed as an argument to every function. `asAnonymous` locally overrides it for one call.

> [!tip] When to Use Reader
> Anytime you have configuration, context, or dependency injection that many functions need but none of them modify — database connections, feature flags, user sessions, logging contexts.
