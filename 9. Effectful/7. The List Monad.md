 ## What Effect Does It Model?

**Nondeterminism** — computations that can produce zero, one, or many results. Each step can branch into multiple possibilities, and `bind` explores all combinations.

## Monad Instance

```lean
instance : Monad List where
  pure x := [x]
  bind xs f := xs.flatMap f
```

- **`pure x`**: a single result `[x]`
- **`bind xs f`**: apply `f` to every element of `xs`, concatenate all results

This means a `do` block over lists acts like **nested loops** that collect all results.

## Example: Cartesian Product

```lean
def pairs (xs : List α) (ys : List β) : List (α × β) := do
  let a ← xs       -- for each a in xs...
  let b ← ys       -- for each b in ys...
  return (a, b)     -- collect the pair
```

```
pairs [1, 2, 3] ["a", "b"]
=  [(1,"a"), (1,"b"), (2,"a"), (2,"b"), (3,"a"), (3,"b")]
```

`let a ← xs` doesn't mean "extract the one value" — it means "for each value in `xs`, continue with that value." This is the nondeterministic interpretation of `bind`.

## Filtering with guard

The `Alternative` instance for lists provides `guard`, which keeps or discards a branch based on a condition:

- `guard true` = `[()]` — branch survives
- `guard false` = `[]` — branch is pruned (no results)

This turns `do` blocks into **list comprehensions**:

```lean
def coprimeProducts (n : Nat) : List Nat := do
  let x ← List.range' 2 n
  let y ← List.range' x (n - x)
  guard (y > x)
  guard (Int.gcd x y == 1)
  return x * y
```

Read as: "for each `x`, for each `y`, keep only the ones where `y > x` and `gcd(x,y) = 1`, then return `x * y`."

> [!tip] Mental Model
> Think of each `let v ← list` as a `for v in list` loop, `guard` as a `continue if not` condition, and `return` as appending to the result list. The monad wires it all together.
