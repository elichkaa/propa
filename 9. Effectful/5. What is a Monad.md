## The Question

`do` notation works for `IO`, for `Except`, for `Option`, and more. These are very different kinds of effects — I/O, failure, nondeterminism. How can the same syntax work for all of them?

## The Answer: A Shared Interface

A **monad** is a type operator `m : Type → Type` that implements two operations:

| Operation | Type | Purpose |
|---|---|---|
| `pure` | `α → m α` | Lift a plain value into the effectful context |
| `bind` | `m α → (α → m β) → m β` | Chain two effectful computations — run the first, feed its result into the second |

`bind` is also written as the infix operator `>>=`.

Any type that implements `pure` and `bind` can use `do` notation. That's the entire mechanism.

## do Notation is Syntactic Sugar

Every `do` block desugars into nested `bind` and `pure` calls:

```lean
-- do notation
do
  let x ← m1
  let y ← m2
  return (x + y)

-- desugars to
m1 >>= (fun x =>
m2 >>= (fun y =>
pure (x + y)))
-- because pure returns x and bind takes x as input and produces results from type x + y
```

Lines whose result is unused (like `IO.println "hello"`) desugar to `let _ ← ...` — the result is bound to a wildcard.

A concrete example with `IO`:

```lean
-- with do
def echoTwice : IO Unit := do
  let stdin ← IO.getStdin
  let line1 ← stdin.getLine
  let line2 ← stdin.getLine
  IO.println (line1 ++ line2)

-- without do (equivalent)
def echoTwice : IO Unit :=
  IO.getStdin >>= (fun stdin =>
  stdin.getLine >>= (fun line1 =>
  stdin.getLine >>= (fun line2 =>
  IO.println (line1 ++ line2))))
```

## Why This Matters

The monad abstraction means:
- **Library authors** define `pure` and `bind` once for their effect type
- **Users** get `do` notation, `←`, `return`, and all monadic combinators (`mapM`, `forM`, etc.) for free
- **Generic code** can be written once and work across all monads

## Intuition for Each Operation

- **`pure x`**: "I already have the answer, just wrap it" — no actual effect happens
- **`bind m f`**: "Run `m`, and *if it produces a result*, feed that result into `f`" — what "if it produces a result" means depends on the monad (succeeds for Except, is non-empty for List, etc.)

> [!summary] One Sentence
> A monad is an interface (`pure` + `bind`) that lets you sequence computations with effects, and `do` notation is just syntactic sugar for calling `bind` repeatedly.
