## The IO Type

`IO α` represents a computation that, when executed by the runtime, may perform I/O side effects and eventually produce a value of type `α`.

- `IO Unit` — performs effects, returns nothing meaningful
- `IO String` — performs effects, returns a `String`

You cannot "extract" a value from `IO` in pure code — this is what enforces the separation.

## do Notation

`do` blocks let you write <mark style="background:#ff4d4f">sequential</mark>, imperative-style code within an effectful context:

```lean
def helloWorld := do
  IO.println "What is your name?"
  let stdin ← IO.getStdin
  let line ← stdin.getLine
  let name := line.trimAscii.toString
  IO.println s!"Hello {name}!"
```

### Three Binding Forms

| Syntax          | Use                                       | Example                      |
| --------------- | ----------------------------------------- | ---------------------------- |
| `let x ← expr`  | Bind result of an **effectful** operation | `let line ← stdin.getLine`   |
| `let x := expr` | Bind result of a **pure** computation     | `let name := line.trimAscii` |
| bare expression | Execute an effect, discard result         | `IO.println "hello"`         |

> [!note] `←` vs `:=`
> This distinction is the core of how `do` blocks separate pure from effectful operations. `←` says "run this effect and give me the result." `:=` says "compute this pure value."

### Return Value

The result of a `do` block is the result of its **last expression**. If the last line is `IO.println ...`, the block has type `IO Unit`.

## Design Principle

Keep `do` blocks **thin** — delegate logic to pure helper functions and only use `do` for the minimal effectful "glue."

```lean
-- Pure function, easy to test
def greet (name : String) : String :=
  s!"Hello {name}!"

-- Effectful shell, as thin as possible
def main : IO Unit := do
  let name ← readName
  IO.println (greet name)
```
