## When Properties Fail

`plausible` doesn't just say "failed" — it produces a **concrete counterexample** showing exactly which input violates the property.

### Example: Broken `reverse`

```lean
def reverse (xs : List α) : List α :=
  match xs with
  | [] => []
  | x :: xs => [x] ++ reverse xs    -- bug: appends to front, not back
```

Testing `reverse (xs ++ ys) = reverse ys ++ reverse xs`:

```
Found a counter-example!
xs := [0]
ys := [1]
issue: [0, 1] = [1, 0] does not hold
```

The counterexample immediately shows that `reverse [0, 1]` returns `[0, 1]` instead of `[1, 0]` — the function isn't actually reversing.

### Example: Quicksort losing duplicates

```lean
partial def quicksort [Ord α] (list : List α) : List α :=
  match list with
  | [] => []
  | pivot :: tail =>
    let left := tail.filter (fun x => compare x pivot |>.isLT)
    let right := tail.filter (fun x => compare x pivot |>.isGT)  -- bug: isGT drops equals
    left.quicksort ++ [pivot] ++ right.quicksort
```

Testing the permutation property:

```
Found a counter-example!
xs := [0, 1, 0, 0, -1, 1, ...]
x := 0
issue: 1 = 6 does not hold
```

The sorted list has 1 occurrence of `0` but the input has 6 — duplicates equal to the pivot are dropped by `isGT`. Fix: use `isGE` (greater or equal) or partition into `< pivot` and `≥ pivot`.

## The Debugging Workflow

1. **Write the property** — forces you to define what "correct" means
2. **Run `plausible`** — generates random inputs automatically
3. **Read the counterexample** — pinpoints the exact failing case
4. **Fix the bug** — guided by the concrete failure
5. **Re-run** — check that the fix works and doesn't break other properties

> [!tip] Multiple Properties Catch Different Bugs
> `isSorted` alone wouldn't catch the duplicate-dropping bug — the output was still sorted. `isPermutation` caught it. Always test **multiple independent properties**.
