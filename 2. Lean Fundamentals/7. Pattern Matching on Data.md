## Basic Idea

Pattern matching deconstructs a value by checking which constructor produced it and extracting the fields:

```lean
def isEmpty (l : List α) : Bool :=
  match l with
  | []     => true
  | _ :: _ => false
```

Each branch handles one constructor. `_` is a wildcard that ignores a field.

## Nested Patterns

Patterns can match multiple levels of structure at once:

```lean
def third? (l : List α) : Option α :=
  match l with
  | _ :: _ :: z :: _ => some z
  | _ => none
```

This matches a list with at least three elements and extracts the third, without nested `match` blocks.

## Multi-Value Matching

Match on **multiple values simultaneously** by separating them with commas:

```lean
def xor (b1 b2 : Bool) : Bool :=
  match b1, b2 with
  | true, false  => true
  | false, true  => true
  | _, _         => false
```

Useful when the result depends on the combination of multiple inputs, like zipping two lists:

```lean
def zip (xs : List α) (ys : List β) : List (α × β) :=
  match xs, ys with
  | x :: xs, y :: ys => (x, y) :: zip xs ys
  | _, _ => []
```

## Handling Partiality

Some operations are only meaningful for certain inputs (e.g., head of a non-empty list). Instead of crashing, return `Option`:

```lean
def head? (l : List α) : Option α :=
  match l with
  | []     => none
  | hd :: _ => some hd
```

This makes partiality explicit in the type — callers must handle the `none` case.

> [!summary] Pattern Matching Rules
> - Every constructor of the type must be covered (exhaustiveness)
> - Patterns are tried top to bottom — first match wins
> - Wildcards `_` match anything without binding a name
> - Patterns can be nested arbitrarily deep
