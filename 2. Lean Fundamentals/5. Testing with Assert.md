## Unit Testing

`#assert` checks that an expression evaluates to the expected value at compile time. If it fails, Lean raises an error — the code won't compile.

```lean
#assert (absolute 42) == 42
#assert (absolute (-7)) == 7
-- #assert (absolute (-42)) == 24  -- would fail to compile
```

## Why Test in FP?

Pure functions are especially easy to test because:
- No setup or teardown needed (no state, no mocks)
- Same input always gives same output
- Tests are deterministic — no flakiness from I/O or timing

## Limitations

`#assert` runs at **compile time**, so it can only test pure, terminating expressions. For effectful code (`IO`), you need different approaches (e.g., running the program and checking output).
