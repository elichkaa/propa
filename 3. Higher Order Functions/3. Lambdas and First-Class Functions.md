## What is a Lambda?

A **lambda** (anonymous function) is a function defined inline as an expression, without giving it a name via `def`:

```lean
fun x => x + 1          -- Nat → Nat
fun x y => x + y        -- Nat → Nat → Nat
fun s => s.length        -- String → Nat
```

In Lean, `fun`, `λ`, and `let f x :=` (in let-bindings) are all equivalent syntax for creating lambdas.

## Why Lambdas?

Lambdas are useful when a function is:
- **Short** — not worth a top-level definition
- **Used once** — as an argument to `map`, `filter`, `fold`, etc.
- **Captures local context** — uses variables from the enclosing scope (closures)

### Avoiding Parameter Threading

Without lambdas, helper functions must receive all context as explicit parameters:

```lean
def readFromMemory (memory : Array UInt32) (base wordSize a : Nat) : UInt32 :=
  memory[base + wordSize * a]!

def compute (memory : Array UInt32) (base wordSize x y : Nat) : UInt32 :=
  readFromMemory memory base wordSize x + readFromMemory memory base wordSize y
```

With a lambda, the helper **captures** `memory`, `base`, and `wordSize` from scope:

```lean
def compute (memory : Array UInt32) (base x y wordSize : Nat) : UInt32 :=
  let read := fun a => memory[base + wordSize * a]!
  read x + read y
```

No need to pass the same arguments repeatedly — the lambda closes over them.

## Functions as Values (First-Class)

In functional programming, functions are **values** like any other. They can be:

| Capability | Example |
|---|---|
| Assigned to variables | `let f := fun x => x + 1` |
| Passed as arguments | `List.map f myList` |
| Returned from functions | `def make : Nat → (Nat → Nat) := fun n => (fun x => x + n)` |
| Stored in data structures | `structure Utils where hash : String → UInt64` |

### Example: Function Factory

A function can **return different functions** based on configuration:

```lean
def hashFunction (config : String) : String → UInt64 :=
  if config == "simple"
  then fun s => s.length.toUInt64
  else fun s => s.hash
```

The returned lambda persists and can be called later — it remembers the choice made at construction time.

### Example: Bundling Functions in a Structure

```lean
structure Utilities where
  hashFunc    : String → UInt64
  encryptFunc : String → List Char
  decryptFunc : List Char → String

def myUtilities (key : String) : Utilities :=
  { hashFunc    := hashFunction "complex"
  , encryptFunc := fun s => (s.toList ++ key.toList).reverse
  , decryptFunc := fun s => s.drop key.length |>.reverse.toString }
```

Construct once with the key, then use `myUtilities.encryptFunc` and `myUtilities.decryptFunc` without mentioning the key again.

## Lean Shorthand Syntax

| Syntax | Meaning |
|---|---|
| `fun x => x + 1` | Full lambda |
| `(· + 1)` | Shorthand: `·` marks the argument |
| `(!·)` | Shorthand for `fun x => !x` |
| `(·, ·)` | Shorthand for `fun x y => (x, y)` |

> [!summary]
> Lambdas let you define functions as values — inline, capturing context, passed around freely. Combined with higher-order functions like `map`, `filter`, and `fold`, they are the primary mechanism for abstraction in functional programming.
