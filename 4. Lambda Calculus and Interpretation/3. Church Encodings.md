## The Core Idea

Lambda calculus has no built-in data types. Instead, data is represented as **functions** — specifically, as **fold functions** for the type. A Church-encoded value *is* its own eliminator: you give it the "what to do for each constructor" arguments, and it picks the right one.

> [!important]
> Fold is a value. A Church-encoded datum doesn't *have* a fold — it *is* the fold.

## Recipe: From Constructors to Church Encoding

1. List the type's constructors
2. Each constructor becomes a parameter of the fold
3. The encoded value is a function that accepts all those parameters and calls the one corresponding to which constructor built it

## Booleans

Constructors: `true`, `false` — neither takes arguments.

```lean
def Bool := ∀ {β}, β → β → β

def true  : Bool := fun ftrue _      => ftrue
def false : Bool := fun _     ffalse => ffalse
```

A Church boolean is a **two-way choice**: give it two options, it picks one.

```lean
#eval true  "yes" "no"   -- "yes"
#eval false "yes" "no"   -- "no"
```

### Boolean Operations

Defined purely with application — no pattern matching, no built-in `if`:

```lean
def not (b : Bool) : Bool := fun t f => b f t
def and (b1 b2 : Bool) : Bool := fun t f => b1 (b2 t f) f
def or  (b1 b2 : Bool) : Bool := fun t f => b1 t (b2 t f)
```

`not` works by swapping the two arguments before passing them to `b`.

## Natural Numbers

Constructors: `zero` (no args), `succ` (one recursive arg).

```lean
def Nat := ∀ {β}, β → (β → β) → β

def zero : Nat := fun fzero _     => fzero
def succ (n : Nat) : Nat := fun fzero fsucc => fsucc (n fzero fsucc)
```

A Church numeral `n` applies `fsucc` exactly `n` times to `fzero`:

```
zero  fz fs  =  fz
one   fz fs  =  fs fz
two   fz fs  =  fs (fs fz)
three fz fs  =  fs (fs (fs fz))
```

### Converting to Lean Nat

```lean
def toNat (n : Nat) : _root_.Nat := n 0 (· + 1)
```

### Arithmetic

```lean
def add (m n : Nat) : Nat := fun fz fs => m (n fz fs) fs
def mul (m n : Nat) : Nat := fun fz fs => m fz (fun acc => n acc fs)
```

`add m n`: start from `n` (which already applies `fsucc` n times to `fzero`), then apply `fsucc` m more times.

## Pairs

Constructor: `pair a b` — one constructor, takes two values.

```lean
def Pair (α β : Type) := ∀ {γ}, (α → β → γ) → γ

def mkPair (a : α) (b : β) : Pair α β := fun f => f a b
def fst (p : Pair α β) : α := p (fun a _ => a)
def snd (p : Pair α β) : β := p (fun _ b => b)
```

A Church pair holds two values and hands them to whatever function you provide.

## Summary Table

| Type | Constructors | Church type | Intuition |
|---|---|---|---|
| Bool | `true`, `false` | `β → β → β` | Two-way choice |
| Nat | `zero`, `succ` | `β → (β → β) → β` | Count: apply step n times |
| Pair | `pair a b` | `(α → β → γ) → γ` | Holds two values, hands them to you |

> [!summary]
> Church encoding represents data as its fold function. Each value "knows" which constructor it came from by selecting the corresponding parameter when called. This demonstrates that lambda calculus — with only variables, abstraction, and application — can encode arbitrary data and computation.
