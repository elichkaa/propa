## What It Is

The lambda calculus is a minimal formal system where **everything is a function**. It has only three constructs:

```
expr ::= x               (variable)
       | λ x. expr       (function abstraction)
       | expr expr       (function application)
```

That's the entire language — no numbers, no booleans, no `if`, no loops. Yet it can express any computation a Turing machine can.

## The Church-Turing Equivalence

Alonzo Church introduced the lambda calculus in 1936. Alan Turing independently introduced Turing machines the same year. They then proved the two systems are **equivalent in computational power**.

| Model | Perspective | Core operation |
|---|---|---|
| Lambda calculus | Everything is a function | Function application |
| Turing machine | Everything is state + tape | State transition |

This means: any program writable in an imperative language has an equivalent purely functional formulation, and vice versa. Functional and imperative programming are equally powerful — they just model computation differently.

## Writing Lambda Calculus in Lean

Lambda calculus expressions can be represented as an inductive type:

```lean
inductive Expr where
  | var (name : String)
  | lambda (name : String) (body : Expr)
  | apply (func : Expr) (arg : Expr)
```

But since Lean has native lambdas, we can write lambda calculus programs directly:

```lean
def id    : α → α         := fun x => x
def first : α → β → α     := fun x y => x
def compose (f : β → γ) (g : α → β) : α → γ := fun x => f (g x)
```
